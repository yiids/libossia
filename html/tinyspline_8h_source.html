1291  /******************************************************************************
1292  * *
1293  * :: Transformation functions *
1294  * *
1295  * TinySpline is a library focusing on transformations. That is, most *
1296  * functions are used to transform splines by modifying their state, e.g., *
1297  * their number of control points, their degree, and so on. Accordingly, each
1298  ** transformation functions specifies an input and output parameter (along *
1299  * with the other parameters required to calculate the actual
1300  *transformation). * By passing a different pointer to the output parameter,
1301  *the transformation * result is calculated and stored without changing the
1302  *state of the input. * This is in particular useful when dealing with
1303  *errors as the original state * will never be modified. For instance, let's
1304  *have a look at the following * code snippet: *
1305  * *
1306  * tsBSpline in = ... // an arbitrary spline * tsBSpline out; //
1307  *result of transformation *
1308  * *
1309  * // Subdivide 'in' into sequence of bezier curves and store the result
1310  **
1311  * // in 'out'. Does not change 'in' in any way. * tsError err =
1312  *ts_bspline_to_beziers(&in, &out); * if (err !=
1313  *TS_SUCCESS) { *
1314  * // fortunately, 'in' has not been changed * } *
1315  * *
1316  * Even if 'ts_bspline_to_beziers' fails, the state of 'in' has not been *
1317  * changed allowing you to handle the error properly. *
1318  * *
1319  * Unless stated otherwise, the order of the parameters for transformation *
1320  * functions is: *
1321  * *
1322  * function(input, [additional_input], output, [additional_output]) *
1323  * *
1324  * 'additional_input' are parameters required to calculate the actual *
1325  * transformation. 'additional_output' are parameters storing further result.
1326  **
1327  * *
1328  * Note: None of TinySpline's transformation functions frees the memory of
1329  *the * output parameter. Thus, when using the same output parameter multiple
1330  ** times, make sure to free memory before each call. Otherwise, you will *
1331  * have a bad time with memory leaks: *
1332  * *
1333  * tsBSpline in = ... // an arbitrary spline * tsBSpline
1334  *out; // result of transformations *
1335  * *
1336  * ts_bspline_to_beziers(&in, &out); // first transformation *
1337  * ... // some code *
1338  * ts_bspline_free(&out); // avoid memory leak. *
1339  * ts_bspline_tension(&in, 0.85, &out); // next transformation *
1340  * *
1341  * If you want to modify your input directly without having a separate
1342  *output, * pass it as input and output at once: *
1343  * *
1344  * tsBSpline s = ... // an arbitrary spline *
1345  * tsReal *knots = ... // a knot vector *
1346  * *
1347  * ts_bspline_set_knots(&s, knots, &s); // copy 'knots' into 's' *
1348  * *
1349  * Note: If a transformation function fails *and* input != output, all fields
1350  ** of the output parameter are set to 0/NULL. If input == output, your *
1351  * input may have an invalid state in case of errors. *
1352  * *
1353  ******************************************************************************/
